# 1. Обновлённое ядро — сделать главной базовой статьёй

Это должна быть статья уровня:

**TMI Core 0.2: Automatic Interfaces and the Primary Interface Axiom**  
по-русски:  
**Ядро ТМИ 0.2: автоматические интерфейсы и аксиома первичного интерфейса**

Её задача — зафиксировать фундамент:

\[
N_0\xrightarrow{I_0}\Delta_0\to(A,\neg A)
\]

\[
Valid(I)\Rightarrow Auto(I)
\]

\[
Auto(I)\not\Rightarrow AlwaysAct(I)
\]

\[
Der(I,I_0)\Rightarrow CondAuto(I)
\]

Главная мысль статьи:

> Действительный интерфейс автоматичен по определению, но автоматичность не означает постоянную активность. Интерфейс сам выполняет свою функцию при наличии допустимого входа.

Это **не физическая статья**. Это онтологико-математическое ядро ТМИ.

# 2. TMI-Physics 2.2 — сделать физическим модулем, который ссылается на ядро

Вторая статья:

**Automatically Structured Interface Fields in TMI-Physics 2.2**  
по-русски:  
**Автоматически структурированные интерфейсные поля в TMI-Physics 2.2**

Её задача — показать физическое следствие из обновлённого ядра:

\[
Valid(I)\Rightarrow Auto(I)
\]

поэтому:

\[
\Xi_I\in\mathfrak I_{phys}
\Rightarrow
Auto(\Xi_I)
\Rightarrow
\Xi_I^{auto}
\]

И дальше:

\[
g
\Rightarrow
Adm_c(g)
\Rightarrow
\mathcal H_I(g)
\Rightarrow
\Psi_I
\Rightarrow
Dist_{poss}(\Psi_I;g)
\Rightarrow
\Xi_I^{auto}
\Rightarrow
Struct_I
\]

Главная мысль статьи:

> Если интерфейсное поле является действительным физическим интерфейсом, то его автоматичность не добавляется извне, а следует из определения интерфейса. Поэтому автоматически структурированное интерфейсное поле становится естественным физическим объектом внутри TMI-Physics.
